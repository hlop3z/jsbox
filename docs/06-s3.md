# 6. `s3` вЂ” Signed Upload & Download Links рџ”—

`s3` makes a **special link** that lets a web page upload a file straight to your
storage bucket (S3, Cloudflare R2, MinIO, Backblaze B2вЂ¦) вЂ” without the file ever
passing through jsbox.

## The big idea рџ’Ў

The robot does **not** touch your files. It only does some quick math to make a
**signed link** (a normal web address with a secret signature on the end). You give
that link to your web page, and the browser uploads the file directly. Fast, and the
robot never sees the bytes.

The robot never connects to your storage вЂ” it just does quick math and hands back a
link. It does **no file reading or writing** at all.

> рџ›ЎпёЏ **Safety rules (same as `api`).** The `endpoint` must start with `http://` or
> `https://` вЂ” never `file://`. And it must be a **public** address: `localhost`,
> `127.0.0.1`, and private network addresses (`10.x`, `192.168.x`, вЂ¦) are **blocked**,
> so a link can never point at something local or internal. Your script can't change
> the `endpoint` either вЂ” only the `config` you send can.

## Turn it on first рџ”‘

Give the robot your bucket details with `config.s3`:

```json
{
  "config": {
    "s3": {
      "endpoint": "https://ACCOUNT.r2.cloudflarestorage.com",
      "region": "auto",
      "bucket": "uploads",
      "access_key": "AKID...",
      "secret_key": "SECRET...",
      "path_style": false
    }
  }
}
```

| Setting       | What it means                                                     | Default    |
| ------------- | ---------------------------------------------------------------- | ---------- |
| `endpoint`    | Your storage address, starting with `http://` or `https://`      | (must set) |
| `region`      | The storage "region" name (R2 uses `auto`)                       | (must set) |
| `bucket`      | The bucket (folder) name                                         | (must set) |
| `access_key`  | Your storage username                                            | (must set) |
| `secret_key`  | Your storage password                                            | (must set) |
| `path_style`  | `true` for MinIO/local; `false` for AWS-style `bucket.host`      | `false`    |
| `expires`     | How long a link works, in seconds                               | `900`      |
| `max_expires` | The longest a link may ever last (7 days)                       | `604800`   |

No `config.s3` в†’ `s3` is turned off.

> **Which `path_style`?** MinIO and most self-hosted stores need `true`. AWS S3 and
> Cloudflare R2 use the default `false`.

## Make an upload link: `s3.presignPut` в¬†пёЏ

```js
function handler(ctx) {
  var put = s3.presignPut({
    key: "uploads/" + ctx.filename, // where to save it in the bucket
    expires: 300, // link works for 5 minutes
  });
  // put = { url: "https://...&X-Amz-Signature=...", method: "PUT", expires: 300 }
  return json({ uploadTo: put.url }, null);
}
```

Your web page then uploads the file with that link:

```js
// in the browser:
await fetch(put.url, { method: "PUT", body: theFile });
```

## Make a download link: `s3.presignGet` в¬‡пёЏ

```js
var get = s3.presignGet({ key: "uploads/photo.jpg" });
// get.url is a short-lived link to view/download the file
```

## The general form: `s3.presign`

```js
s3.presign({ method: "PUT", key: "uploads/x.txt", expires: 120 });
// method can be "PUT", "GET", "HEAD", or "DELETE" (defaults to "PUT")
```

## When something goes wrong

`s3.presign*` **throws an error** if something is missing (like an empty `key`).
Catch it with `try/catch`:

```js
function handler(ctx) {
  try {
    var put = s3.presignPut({ key: ctx.key });
    return json({ url: put.url }, null);
  } catch (e) {
    return json(null, { message: e.message });
  }
}
```

## It shows up on the receipt рџ§ѕ

Each signed link is listed in `meta.s3_requests` (the method and how long it lasts).
